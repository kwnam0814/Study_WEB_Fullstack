# CSS

## 반응형 웹 이해

과거에는 웹은 PC의 영역이었지만,

요즘에는 스마트폰, 테블릿 등 다양한 디바이스에서 사용하는 경우가 많다.

최근 웹 개발 트렌드는 React, Vue, Angular 같은 웹 프레임워크이다.

반대로 제이쿼리에 대한 인기는 점차 식어가고 따라서 부트스트랩의 인기도 점점 낮아지고 있다.

개발자는 반응형 웹을 이해하고 처리할 수 있어야 한다.

`반응형 웹`

디바이스 화면 사이즈에 따라 반응하여 다르게 보여주는 기술.

`적응형 웹`

디바이스가 모바일인지, PC인지에 따라 다른 페이지를 보여주는 것.

네이버가 대표적으로 적응형 웹을 채택하고 있다.

`픽셀과 뷰포트`

픽셀 : 화면을 표시하는 최소단위

![픽셀](./css_pixel.png)

반응형에서 가장 큰 이슈는 디바이스의 해상도 크기이다.

같은 16px도 해상도에 따라서 사이즈는 다르게 보일 수 있다.

이런 문제에 대응하기 위해 `뷰포트`가 등장했다.

뷰포트는 우리 눈에 보이는 화면 표시 영역을 의미한다.

모바일 환경에서는 픽셀을 그대로 사용하지 않는다. `가상 뷰포트` 라는 개념을 사용한다.

대부분 너비 980px의 가상 뷰포트를 기준으로 렌더링된다.

실제 모바일 폰이 1440, 1242, 1080, 720 .. 등 다양한 픽셀을 가지고 있더라도 마치 980px인것처럼 보이게 만드는 것이다.

하지만 이마저도 가독성이 떨어진다는 단점이 있다.

각 디바이스별로 너비에 알맞게 웹페이지를 렌더링할 수 있도록 device-width가 정의된다.

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

위 메타 태그를 정의하지 않는다면 기본적으로 980px를 기준으로 렌더링한다.

`관련 속성`

- width: 뷰포트 너비 (device-width)
- initial-scale: 뷰포트 초기 배율 (기본값: 1)
- height: 뷰포트 높이
- user-scale: 확대, 축소 가능 여부
- maximum-scale & minimum-scale : 뷰포트 최대&최소 배율
